# The Coffee Handbook

A browser-based market-reading and note-taking tool built for a green coffee trading desk. It turns the daily routine of reading the market into a structured, repeatable process, and keeps a living record of those reads so they can be compared over time.

**Live site:** https://mansoursilvio-star.github.io/The-coffee-handbook/

---

## What it does

The handbook is a single-page tool with three parts.

**Note-taking.** A guided flow for writing up a market read. It walks through the things that actually move coffee (price action, weather and multi-origin risk, certified stocks, the spreads, the macro backdrop) and produces a clean, shareable note at the end. Notes can be saved as daily or weekly, and each saved note keeps its own date. Saving an open note updates it in place rather than creating a duplicate.

**Find an analog.** A historical-comparison engine. Given today's read, it ranks past notes by how similar they are, weighted heavily on the price level, with optional tickable filters (overall lean, Brazil and Vietnam stance, the main driver, note type, date range). The idea is simple: when you are trying to read today, it helps to see the days that looked like this one and what happened next.

**Learn more.** A reference section covering the mechanics behind each question in the note flow, plus a condensed guide to the things worth knowing about the coffee market.

## Prices

The tool tracks the **full futures curve** — every listed contract month — for both arabica (KC) and robusta (RC/RM).

**Both pull automatically from Barchart.** A scheduled job runs each weekday morning, scrapes the futures-prices table for each coffee root from Barchart, and writes every listed month (with last, prior settle, change, OHLC, volume and open interest) to a small data file that the handbook reads. In the handbook, one click fills the front-month price into a note, or you can expand the full curve and pick any month.

**How the scrape works.** Barchart renders its futures-prices table from an internal JSON endpoint that is guarded by a cross-site token: the site sets an `XSRF-TOKEN` cookie on first page load and expects it echoed back in a request header. The script loads the page once to collect that cookie, then calls the endpoint with the token to pull the whole curve. No API key, no paid feed, no third-party Python packages.

**Paste box still there as a backup.** If Barchart blocks the runner on a given day, KC falls back to a free front-month source (Yahoo) so the file is never wiped to empty, and you can always paste a row from a market data page. A built-in parser reads pasted rows and pulls out the month, price, and change automatically.

The two coffee roots on Barchart are `KC` (arabica, ICE US, US cents/lb) and `RM` (robusta, ICE London, USD/tonne).

## Profiles and privacy

Notes are saved in the browser, on the device they were written on. They never leave the machine and are not stored on any server. A local profile feature lets more than one person keep separate note sets on a shared device. It is a convenience label, not a secure login. For moving notes between your own devices, the tool can export and re-import the full history as a file.

## How it is built

- **Front end:** a single self-contained HTML file (HTML, CSS, and JavaScript in one file). No framework, no build step. State is kept in the browser using local storage.
- **Hosting:** GitHub Pages (static hosting). Hosting the file as a real website is also what makes the saving reliable, since browsers block local storage on files opened directly from disk.
- **Price automation:** a small Python script (`fetch_prices.py`) with no external dependencies (standard library only), run on a schedule by a GitHub Actions workflow (`.github/workflows/update-prices.yml`). The script scrapes the Barchart futures-prices table for both coffee roots and writes the full curve to `prices.json`, which the handbook fetches and reads.

## Files in this repo

| File | Purpose |
|------|---------|
| `index.html` | The handbook itself. |
| `fetch_prices.py` | Scrapes the full KC and RC futures curve from Barchart each morning. |
| `.github/workflows/update-prices.yml` | Schedules and runs the fetch. |
| `prices.json` | Generated by the script. Holds the latest prices. |

## Running the price job by hand

The fetch runs automatically on weekday mornings, but it can also be triggered from the Actions tab in GitHub (the workflow has a manual run button). After it runs, `prices.json` updates and the handbook picks up the new prices on the next load.
